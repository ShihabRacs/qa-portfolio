Test Case ID: TC_LOGIN_001
Title: Valid login with correct credentials
Preconditions: User account exists with email test@example.com / password Test@123
Steps:
  1. Navigate to https://example.com/login
  2. Enter "test@example.com" in the Email field
  3. Enter "Test@123" in the Password field
  4. Click the "Login" button
Expected Result: User is redirected to the dashboard and sees "Welcome" message

Test Case ID: TC_LOGIN_002
Title: login with incorrect email
Preconditions: User account does not exist with email test@example.com / password Test@123
Steps:
  1. Navigate to https://example.com/login
  2. Enter "test@example.com" in the Email field
  3. Enter "Test@123" in the Password field
  4. Click the "Login" button
Expected Result: User is shown some incorrect message "Incorrect email/password, if you are a new user, you can use sign up below".
Note: System must NOT reveal whether the email or password 
specifically is wrong. A generic message prevents credential 
enumeration attacks.

Test Case ID: TC_LOGIN_010
Title: login with incorrect email multiple times
Preconditions: User account does not exist with email test@example.com / password Test@123
Steps:
  1. Navigate to https://example.com/login
  2. Enter "test@example.comasdf" in the Email field
  3. Enter "Test@123" in the Password field
  4. Click the "Login" button
  5. repeate steps 2 to 4 for 5 times
Expected Result: User should be blocked for some time to use the login form. the form should say "Too many invalid attempts".
Note: Also verify the lockout is time-based (e.g. unlocks after 15 minutes) 
or requires manual admin unlock, depending on the system's security policy.
